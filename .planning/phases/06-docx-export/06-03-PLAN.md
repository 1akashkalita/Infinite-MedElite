| Boundary | Description |

| browser client → export route | `ExportControls` is a `"use client"` component; it must call the server route over HTTP and must NOT import the server-only `docx` builder. The trust boundary is the `fetch` to `/api/export/{pdf|docx}` (validated server-side in Plan 06-02). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-08 | Elevation of Privilege | `docx`/`ReportDocx` reaching the client bundle via ExportControls | mitigate | ExportControls imports only `ReportViewModel` as a TYPE; never `docx`/`Packer`/`ReportDocx`/`ReportPDF`. The Task-1 grep guard + the Task-2 `next build` (verify:full) fail fast if a server-only module leaks into the client bundle (Pitfall 3 / PITFALLS #4). |
| T-06-09 | Information Disclosure | export failure surfaced to the user | mitigate | D-08: on any export failure the component shows a fixed UI-authored string ("Couldn't generate DOCX — try again."), never the raw server response or Zod internals; the error stays inline (role=alert) and is never routed through the CMS-lookup ErrorBanner. |
| T-06-10 | Denial of Service | repeated/concurrent export clicks | accept | A single shared in-flight lock (`loading`) disables the button and toggle while a request is pending; switching format is only allowed when idle. Low-value internal take-home tool, stateless route — no rate limiting needed beyond the in-flight lock. |
| T-06-SC | Tampering | npm/pip/cargo installs | mitigate | No installs in this plan. No `[ASSUMED]`/`[SUS]` package introduced. |
