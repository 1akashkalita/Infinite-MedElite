<objective>
Build the two server-side foundations of the `.docx` export: (1) generalize the existing `slugFilename` helper to accept an extension parameter so both export routes share the same injection-safe sanitization (D-13), and (2) create `ReportDocx.ts`, the server-only `docx`-library twin of `ReportPDF.tsx` that produces the Word `Document` from a `ReportViewModel`.

Purpose: This plan delivers everything the `.docx` route needs to exist (the document builder + the filename helper) without yet wiring the HTTP boundary. It is fully testable in isolation (slug unit tests + typecheck of the builder), keeping risk low before the route slice consumes it.

Output: A generalized `slugFilename(name, ccn, ext)`, extended `slug.test.ts`, and a new `src/lib/docx/ReportDocx.ts` exporting `buildReportDocx(vm): Document`.
</objective>
