## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→API (untrusted POST body) | The browser POSTs a `ReportViewModel` JSON. It is untrusted and MUST be re-validated server-side (standing rule #4) before anything is built or rendered into headers. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04 | Tampering | POST body → buildReportDocx | mitigate | `ReportViewModelSchema.safeParse(body)` rejects any malformed/forged shape with a 400 before the builder or Packer runs. `await request.json()` is wrapped in try/catch so a non-JSON body yields a clean 400, never a raw 500. |
| T-06-05 | Information Disclosure | 400 response body | mitigate | D-05/D-12 clean-envelope discipline: the 400 body is always exactly `{ error: { kind: "invalid_request", message: "Invalid report data." } }`. Zod `.issues`/`.paths`/`.codes` never reach the client. Asserted by the "does not leak Zod internals" test. |
| T-06-06 | Tampering | `displayName`/`ccn` → Content-Disposition filename | mitigate | The route passes both through `slugFilename` (allowlist-sanitized in Plan 06-01) before composing the header. CRLF/quote/slash payloads cannot survive into Content-Disposition (T-04-03/CR-01). |
| T-06-07 | Information Disclosure | `careCompareUrl` in the embedded Word hyperlink | accept (upstream-mitigated) | `ReportViewModelSchema` already constrains `careCompareUrl` to `https://www.medicare.gov`; a non-medicare URL fails safeParse → 400 before the doc is built. No new control. |
| T-06-08 | Elevation of Privilege | `docx`/`buildReportDocx` reaching the client bundle | mitigate | The builder + `Packer` are imported only by this server route. `next build` (verify:full — run at the phase gate / Plan 03) fails fast if either reaches a `"use client"` module. |
| T-06-SC | Tampering | npm/pip/cargo installs | mitigate | No installs — `docx@9.7.1` already present (06-RESEARCH Package Legitimacy Audit: approved). No `[ASSUMED]`/`[SUS]` package introduced. |
</threat_model>
