## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser client → POST /api/export/pdf | The button serializes the in-state `vm` and POSTs it; the server re-validates with `ReportViewModelSchema.safeParse` (Plan 04-01) — the client is never trusted as the validation authority. |
| @react-pdf/renderer (server-only) → client bundle | A build-time boundary: the package must never cross into the `"use client"` bundle, or `next build` breaks and the live deploy fails. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Tampering | client-supplied POST body (in-state vm) | mitigate | The button is a convenience trigger only; the route's `ReportViewModelSchema.safeParse` (Plan 04-01) is the authoritative gate. A tampered body fails validation server-side → 400; the button surfaces a generic inline retry error (no internals leaked). |
| T-04-06 | Elevation of Privilege / Build integrity | DownloadPdfButton.tsx and SnapshotApp.tsx (client bundle) | mitigate | Enforce that neither file imports `@react-pdf/renderer` nor `ReportPDF` (source assertions in both tasks); `npm run verify:full` (`next build`) is the gate that fails if the server-only package leaks into the client bundle (PITFALLS #4). |
| T-04-07 | Information Disclosure | inline export-error copy (D-08) | accept (already mitigated) | The button renders a fixed UI-authored string ("Couldn't generate PDF — try again."), never the raw server response or any Zod/internal detail — same discipline as CCNSearchBar's getErrorPresentation copy. |
| T-04-SC | Tampering | npm/pip/cargo installs | n/a | No new packages installed this phase (04-RESEARCH.md Package Legitimacy Audit). No install task exists. |

No `high`-severity threats. The only new client-side surface is a button that triggers an already-validated server route; the build gate guards the server-only-package boundary.
